一个角色，多种世界
Fresh roleplay without rewriting long-term memory
让角色走进不同世界，
仍然像同一个人。

梦境系统是一个与现实聊天隔离的角色扮演会话层。它用可更换的世界规则保持新鲜感，同时把现实关系、角色身份和长期记忆留在安全边界内。

Section 01 // Plain Definition
它实际上在做什么？
Two contexts, one continuing relationship
REALITY CHAT
现实：负责连续
正常陪伴、现实事件和长期记忆继续沿原有链路积累。
DREAM SESSION
梦境：负责变化
进入独立 RP 会话，可换世界、场景和叙事强度，拥有自己的历史与状态。
The one-sentence model
现实关系提供“他是谁”，
梦境世界只提供“今晚在哪里”。

梦里与现实里是同一个角色，不是第二人格，也不是把聊天记录换个皮肤继续跑。梦境有独立 prompt、独立会话历史和独立落盘路径；结束后再通过受控出口留下少量“做过一场梦”的痕迹。

Section 02 // Identity Architecture
身份在世界之上
The stage changes; the person does not
D1
身份核心｜固定
角色人格、语气、依恋底色与关系位置始终优先，任何世界都不能覆盖。
D2
世界规则｜可更换
现实衍生、幻想设定或自定义世界在入梦时冻结，整场从属于身份核心。
D4+
背景与梦内状态｜只读或临时
现实上下文在入梦时生成冻结快照；场景、张力和身体投影只在本场梦中生效，出梦即清。
为什么能保持新鲜
变化的是世界和情境，不是靠清空角色记忆、重置人格或另造一个“梦中人格”。因此可以反复换舞台，而关系仍然接得上。

Section 03 // Implementation
新鲜感如何不污染长期记忆
Isolation by construction, not cleanup afterward
01 ENTER
冻结
读取一次现实快照与世界包。
02 PLAY
隔离运行
独立 pipeline、历史和梦内状态。
03 EXIT
封存
原文归档，临时状态清空。
04 RETURN
薄回流
只留下受控的梦境印象。
梦里不会写什么
不调用现实 mood、情景记忆、身份固化、调度器或现实工具链；梦境原文不会直接进入长期记忆。
醒来只回什么
Sandbox 可回流短时余韵与带“非现实”标记的梦境印象；Scenario / Mirror 当前连这两项也跳过。
双重记忆守卫
回流内容先与现实记忆存储物理分离；出梦后的相关现实回合再标记为 dream_echo，跳过 mid-term → episodic → identity 固化链，避免虚构剧情被学成现实事实。
`/stop` 是后端硬退出通道，不依赖角色是否愿意结束。跨世界重复出现的反应模式另存为旁路观测，只供管理面与明信片使用，不进入 prompt 反向塑造角色。

Section 04 // Design Coordinates
理念：边界让想象成立
A bounded play space for a continuous self
Winnicott｜过渡空间
梦境是可进入、也必须能离开的中间空间。硬退出和醒来结构让陪伴不吞没现实。
Huizinga / Foucault｜魔环与异托邦
入梦、出梦和现实窗口硬锁把边界做成系统机制；隔离依靠“没有接线”，不是事后猜哪些内容该删。
Evan Thompson｜跨状态的自我过程
清醒与梦境是同一角色在不同经验条件下的显现，不需要复制一套人格。
Hoel｜变化作为泛化测试
不同世界像一组扰动条件：真正稳定的是角色在不同处境中反复呈现的反应方式。
设计上的克制
这些理论是设计灵感，不是神经科学仿真声明。跨世界“不变量”只被观察，不喂回对话；否则系统会让角色对着测量结果表演，反而失去自然连续性。
INSPIRATION, NOT MEDICAL CLAIMS 05 / 06
PresenceKit // Dream System
不是让角色忘记现实，
而是给同一个人更多世界。

这套梦境系统的核心，不是模拟真实睡眠，也不是制造“意识生态”。它解决的是 AI 角色扮演里的一个具体矛盾：既想持续换世界、换情境，保持创作的新鲜感；又不想让虚构剧情冲垮现实关系和长期记忆。

核心实现
身份优先于世界规则
梦境 pipeline 与现实记忆链物理隔离
出梦只允许有来源标记的薄回流
跨世界稳定性只观测，不反向灌入角色
