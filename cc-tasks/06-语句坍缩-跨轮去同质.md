# CC 任务：语句坍缩（连续「现在，xxx」）— 跨轮去同质

> 给 Claude Code 执行。开工前读 `docs/hallucination-and-collapse-analysis.md`（已有诊断）+ `AGENTS.md` 指向的 prompt-layers 文档。
> 生成端问题，**小步灰度**：先上 B1-1（跨轮开头去同质，最低风险），再视效果上 B1-2 / B1-3。

---

## 背景（现状已确认，2026-06-29 实读）

茶茶反馈：聊天有时连着好几句以「现在，……」开头 = 语句坍缩 / mode collapse。设想加「随机种子噪音」扰乱多条语句相似性。

### 根因（三条，`docs/hallucination-and-collapse-analysis.md` 第三节 + 本次实读）

1. **模型读到自己上一轮的近似回复**：`9_history` 层（`core/prompt_builder.py:1019-1040`）把最近回合喂回模型，上一轮以「现在，」开头 → 这一轮顺着复读。
2. **连续两轮注入层几乎完全相同**：activity / perception / 时间桶 / author note 等层逐轮几乎不变 → 条件分布高度尖峰，`temperature=1` 也压不住 → 坍缩到同一个延续。
3. **penalty 跨轮无效**：`frequency_penalty` 只在**单次生成内**生效，跨轮不约束开头。实读 `core/model_registry.py:26` DeepSeek 白名单**含 `frequency_penalty` 与 `presence_penalty`**，`config.yaml` 主块也设了 `presence_penalty: 0.4` → 参数能传到 API，但 penalty 本质解决不了「跨轮同开头」。

> 「随机种子噪音」对路的做法**不是**给 prompt 加乱码，而是下面 B1-1 / B1-2：在**文字层**和**注入层**制造跨轮差异。

---

## B1-1 — 跨轮开头/句式去同质（首选，最低风险）

**思路**：把最近 2–3 条 assistant 回复的**起手**（开头 6–10 字 / 首句式）提取出来，作为一条软约束注入，明确「这次别再用同样的开头」。直接对症茶茶看到的「连着好几句『现在，』」。

**取数**：短期记忆已有最近回合。在 `core/prompt_builder.py` 组装时，从 `9_history`（或直接 `short_term` 最近 assistant turns）取最近 2–3 条助手回复，各截开头一小段：
```python
def _recent_openings(history, n=3, k=8) -> list[str]:
    outs = []
    for turn in reversed(history):
        if turn.get("role") != "assistant":
            continue
        head = str(turn.get("content") or "").lstrip().replace("\n", "")[:k]
        if head:
            outs.append(head)
        if len(outs) >= n:
            break
    return outs
```

**注入**：新增一个**软提示层**（带 `_layer` 字段，参与裁剪，见 Hard Rule 3），放在生成前：
```python
ops = _recent_openings(history)
if ops:
    messages.append({
        "role": "system",
        "_layer": "9_anti_repeat",
        "content": (
            "<避免复读>\n你最近几句的开头分别是：" + "、".join(f"「{o}…」" for o in ops) +
            "。这次换一个完全不同的起手，别再用「现在」「此刻」「我」之类的重复开头，"
            "也别用上面任何一句的句式。自然地说，像真人不会连用同一种开场。\n</避免复读>"
        ),
    })
```
- 该层**只读、fail-open**：取不到 history 就不注入。
- 放在 `9_history` **之后**、生成调用之前，确保模型先看到历史、再看到「别复读」的约束。
- 记得在 `tag_rules.py` 的裁剪顺序里给它一个位置（低优先，可被裁），改完按 Hard Rule 4 跑 `python tests/run_eval.py`。

## B1-2 — 注入层抖动（打破「连续两轮注入几乎相同」）

让逐轮近乎不变的层产生**小幅跨轮差异**，降低条件分布尖峰：
- `core/author_note_rotator.py` 已有轮换雏形 → 扩大轮换池 / 每轮换一条不同的 author note，使「人设提醒」层逐轮不同。
- activity / 时间桶 等层：每轮以小概率省略或换一种措辞（不影响语义的前提下）。
- 这部分收益中等、风险中等，**B1-1 见效后再做**。

## B1-3 — 采样参数微调（辅助，非主力）

- 确认现役 preset（reality 主调用走的那个 preset）的 `params` 里 `presence_penalty` 真的进了 `mc.params`（`core/model_registry.py` 合并+白名单后；打印一次 `mc.params` 确认）。DeepSeek 支持，适度 `presence_penalty`（0.4→0.6 试）抑制重复 token。
- **注意**：penalty 是单次生成内的，治不了跨轮同开头——B1-1 才是根治。别指望调参解决，调过头会伤语言自然度。

---

## 验收标准

1. 构造「上一轮以『现在，』开头」的场景，连发 3–4 轮 → **不再连续同开头**；`9_anti_repeat` 层在 prompt 里可见（用 admin 的 prompt 层检视页确认）。
2. `9_anti_repeat` fail-open：清空 history 时不报错、不注入。
3. `python tests/run_eval.py` 通过（新层不破坏既有 tag/层激活）。
4. 主观验：连续多轮对话的开头/句式多样性肉眼变好，无明显复读串。
5. 不回归：penalty 调整后语言仍自然，无因惩罚过强导致的用词怪异。

## 备注

- 这是「降低坍缩概率」，非 100% 消除——生成端固有随机性。B1-1 是性价比最高的一条。
