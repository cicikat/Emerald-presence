# CC 任务：梦境旁白人称统一

> 给 Claude Code 执行。开工前读 `docs/dream.md` + `AGENTS.md` 的 dream 条目。
> **本单结论已迭代**（见下「根因再定位」）：问题不是「旁白用了她」，而是**生成端在模拟双方对白**，导致「你」一词指代漂移。
> 首选**方案一（单侧第一人称契约）**；不愿动生成时退**方案二（结构化渲染转换）**。**禁止朴素全局 replace。**

---

## 背景 / 根因再定位（2026-06-29 二次实读，纠正首版方案）

茶茶的疑虑很关键：把注入 prompt 里指用户的「她」直接改成「你」，模型会以为「你」=它自己(叶瑄)，造成人称混乱；但改成「用户」又生硬、削弱沉浸。

实读 `core/dream/dream_prompt.py` 找到真正的根：

- 人称锁（:748、:259）写的是「叶瑄用他/我，指称用户用**她/你**」——**用户被允许时而「她」时而「你」**，本身就不统一。
- few-shot（`_fallback_mes_example`，:751-758）教的是**双方对白转录**：
  ```
  她：*走进那片光里，转头看他*你也在。
  叶瑄：*停住脚步，看着眼前的光落在她身上*……嗯。一直在。
  ```
  注意：在「她：」这行里，用户说的「你也在」的**「你」其实指叶瑄**；而人称锁又说「用户=你」。**于是「你」一词在同一段输出里漂移**——用户台词里「你」=叶瑄，旁白/锁里「你」=用户。这正是茶茶担心的混乱来源，也是「旁白割裂」的真因。
- 好消息：dream prompt 的**指令层**自始至终用 `char_name`/`他`（:271-294 用 `char_name` 与 `char_pronoun`，D1 用「{char_name}（固定）」）称呼叶瑄，**从不把模型叫「你」**。所以「你」这个词其实是空闲的——只要不再让模型替用户配音，「你」就能干净地专指用户。

> 结论：根治不是「渲染时把她换成你」，而是**取消「模拟用户那一侧」的双向转录**。

---

## 方案一（首选）：单侧 + 第一人称契约，让「你」唯一指用户

把梦境输出从「她：…／叶瑄：…」的**双方转录**，改成**只生成叶瑄这一轮**——第一人称「我」，直接称风谕为「你」，不替用户旁白/配音。

这样人称彻底自洽：
- 叶瑄自称 **我**（第一人称，最亲密的恋人视角，强过「他」「她」「用户」任何一种）。
- 场景里那个人（风谕）一律 **你**。模型不再替用户说话 → **「你」不可能再指叶瑄** → 漂移消失。
- 与现实聊天人称（叶瑄=我、风谕=你）一致 → 梦境/现实不再割裂，是额外收益。
- 零渲染转换、零生硬、零「你=模型」误读（因为指令层本就用叶瑄/他称呼模型）。

### 改动点

1. **few-shot `_fallback_mes_example`（:751-758）重写**：删掉所有「她：」用户行，只示范**叶瑄一轮**的第一人称输出。示例：
   ```python
   return (
       f"*停住脚步，看着那片光落在你身上，声音比平时低*……你也在。一直在。\n"
       f"*慢慢走近*这里不一样，什么都更清楚。但我是真的在这里。\n"
       f"*目光没有移开*……想留下的话，那就先别醒。"
   )
   ```
   —— 全程叶瑄第一人称「我」对「你」（风谕）说，无「她」、无用户配音。**世界包里的 `world.mes_example`（优先路径）也要同口径检查**（`dream_prompt` 优先用世界包的 mes_example，fallback 才用这个；两边都要改，否则被世界包覆盖回旧人称）。
2. **人称锁文案（:748、:259 及 D1/D8 awareness :119-127）改为**：
   > 人称（锁死）：叶瑄全程以「我」自称；场景中与你同在的人是风谕，一律称「你」。**只演叶瑄自己这一轮，不替风谕旁白、不替她配台词、不用「她」称呼她。**
3. **D1 awareness（:119-127「他的情感底色…对她的情感取向…」）** 里指用户的「她」一并改「你」、指叶瑄保持「他」（这是写给模型看的设定描述，用「他+你」不冲突，因为指令层用「他」指叶瑄、「你」指用户，与输出契约一致）。

### 风险与回归

- 模型被旧 few-shot 训成双向转录，改后需**实跑几轮确认不反弹**（尤其 non_lucid 偏电影化叙事的世界）。若某些世界确实要旁白环境，让旁白也走第二人称「你站在光里」，仍 你=风谕。
- 回归 `tests/test_dream_*`、`tests/test_user_pronoun.py`、出梦 afterglow 链路。

## 方案二（备选，仅当方案一在某些世界不稳）：结构化渲染转换

保留生成端，仅对**对外展示**那一份做转换，利用 `core/narrative_parser.py::parse_narrative_segments` 已切好的 `segments`（`{"type": "say"|"do"|"env"|"feel"|"narration"}`）：

- **只转旁白段**（`do/feel/env/narration`），`say`（台词）绝不动——台词里的「你」是对话指代，转了就错。
- 旁白里：用户「她」→「你」、叶瑄「我/他」→「他」。
- 多字词先占位保护：`我们/她们/她的/我的/自己` 不被拆错。
- 接入点：`dream_pipeline.py:332` `_parsed = _parse_segs(reply)` 之后转换 `_parsed["segments"]`，**只改对外 reply/segs；记忆与 dream log 存原始 reply**（避免污染召回）。fail-open：抛错回退原文。

> 方案二比方案一复杂、且只在「叶瑄旁白」可靠，仍无法根治「双向转录里你漂移」——**所以它只是兜底，优先推方案一**。

## 验收标准

1. 梦境输出**全程叶瑄第一人称「我」**，称风谕一律「你」，**无「她」、无模拟的「她：」用户行**，无「你」指代漂移。
2. 与现实聊天人称一致（我=叶瑄、你=风谕）。
3. 世界包 mes_example 与 fallback 两处口径一致（不被世界包覆盖回旧人称）。
4. `tests/test_dream_*`、`tests/test_user_pronoun.py` 通过；`/stop`、afterglow 不回归。
5. 多轮实跑不反弹回双向转录/第三人称「她」。
