```python
if "global_proactive_min_gap_hours" in body:
    try:
        hours = float(body["global_proactive_min_gap_hours"])
    except (TypeError, ValueError):
        raise HTTPException(status_code=422, detail="global_proactive_min_gap_hours 必须是数字")
```

## Part 2 — 电脑前端（Emerald-client）：镜像 presence_nag

### 2.1 Rust 命令（`src-tauri/src/lib.rs`，紧挨 `patch_presence_nag` :1598）

```rust
#[tauri::command]
async fn patch_proactive_gap(hours: f64) -> Result<(), String> {
    let cfg = load_cfg()?;                      // 与 patch_presence_nag 同款取 cfg
    let resp = authorized_request(&cfg, http_client()?.put(backend_url(&cfg, "/scheduler/config")))
        .json(&serde_json::json!({ "global_proactive_min_gap_hours": hours }))
        .send().await.map_err(|e| e.to_string())?;
    require_success(resp).await.map(|_| ())
}

// 读当前值（GET /scheduler/config → seconds/3600），供面板打开时回填
#[tauri::command]
async fn get_proactive_gap_hours() -> Result<f64, String> {
    let cfg = load_cfg()?;
    let resp = authorized_request(&cfg, http_client()?.get(backend_url(&cfg, "/scheduler/config")))
        .send().await.map_err(|e| e.to_string())?;
    let resp = require_success(resp).await?;
    let body: serde_json::Value = resp.json().await.map_err(|e| e.to_string())?;
    let secs = body.get("global_proactive_min_gap_seconds").and_then(|v| v.as_f64()).unwrap_or(2700.0);
    Ok(secs / 3600.0)
}
```
（`load_cfg`/`authorized_request`/`backend_url`/`http_client`/`require_success` 按 `patch_presence_nag` 实际用的同名 helper 对齐。）**两个命令都要加进 `generate_handler![]`（:1848-1940）。**

### 2.2 TS 封装（新建 `src/shared/proactiveGap.ts`，镜像 `presenceNag.ts`）

```ts
import { invoke } from '@tauri-apps/api/core';

export async function getProactiveGapHours(): Promise<number> {
  try { return await invoke<number>('get_proactive_gap_hours'); }
  catch { return 0.75; }   // 兜底 45 分钟
}
export async function patchProactiveGapHours(hours: number): Promise<void> {
  await invoke('patch_proactive_gap', { hours });
}
```

### 2.3 UI（`src/windows/chat/ChatWindow.tsx` 的 `PreferencesPanel`「对话」分区）

在 presence_nag 那条 `PrefRow`（:441）附近加一条：
```tsx
<PrefRow label="主动消息最小间隔" hint="他每隔至少这么久才会主动找你一次 · 当前 {proactiveGapHours} 小时">
```

## 验收标准

1. 电脑端「偏好 → 对话」出现「主动消息最小间隔 = N 小时」，打开时显示后端当前值。
2. 改成例如 2 小时 → `config.yaml` 的 `global_proactive_min_gap_seconds` 变 7200、`reload_config()` 当场生效（可临时设 0.5h 验证主动消息间隔缩短，再调回）。
3. 越界（0 / >24 / 非数字）后端 422，前端提示不崩。
4. presence_nag 等原有「对话」分区设置不回归。
5. 手机端未改动。
