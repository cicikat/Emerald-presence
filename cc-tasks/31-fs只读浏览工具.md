# Brief 31 · fs 只读浏览工具(fs_list / fs_read)

> 目标:让角色能"自己翻电脑"——列目录、读文件,范围严格限于 config 声明的允许根目录,
> **只读**。写入不新增任何入口(已有 `write_toy_file` 沙盒,维持唯一写出口)。
> 暴露方式与 MCP 同策略:**只经 tool loop(路径C)**,角色卡 `tool_categories` 含 `"fs"` 才可见。
>
> 依赖:Brief 28(loop)、29(per-char tool_categories)已合并。与 30 无依赖关系。

## 2. 配置

```yaml
fs_access:
  enabled: false                  # 总开关,默认关
  allow_roots:                    # 只读允许根,绝对路径,用户手填
    - "D:/some/dir"
  deny_names:                     # 命中即拒(对路径任一段做大小写不敏感子串匹配)
    - "secrets" 
    - ".env"
    - ".git"
    - "node_modules"
    - "__pycache__"
    - "config.yaml"
    - "token"
  max_read_chars: 4000            # 单次读取截断
  max_list_entries: 100           # 单次列目录条数上限
```

默认 deny_names 写死在代码里作为**不可移除的底线集**,config 的 deny_names 只能追加不能删除
(防手滑清空)。`data/` 目录(本项目沙盒)永远隐式拒绝,即使被 allow_roots 包含。

## 3. 工具规格(`core/tools/fs_browse.py`,新文件)

### `fs_list`

- 参数:`path`(可省,省略时返回 allow_roots 列表本身,作为角色的"入口地图")、`depth`(1 或 2,默认 1)。
- 返回:文本列表,目录带 `/` 后缀,文件带大小(KB);超 `max_list_entries` 截断并注明。
- description 写清"你可以浏览这些目录,想看内容用 fs_read"。

### `fs_read`

- 参数:`path`(必填)。
- 只读文本类:按扩展名白名单(txt/md/py/js/ts/json/yaml/toml/csv/log/html/ini 等,代码内维护),
  其他扩展名返回"这是二进制/不支持的文件类型,大小 X"。UTF-8 优先,失败尝试 gbk,再失败按二进制拒。
- 超 `max_read_chars` 截断并注明"文件共 X 字,已截断,可指定更精确的问题"。v1 不做分页偏移。

### 共同守卫(每次调用,顺序执行)

1. `enabled` 为假 → 返回"文件浏览未开启"。
2. `Path(path).resolve()`(realpath,吃掉 `..` 与软链)后必须是某个 `allow_roots` resolve 结果的子路径,否则拒绝。
3. resolve 前后的路径逐段过 deny_names(底线集 ∪ config 追加集),命中拒绝。
4. 软链文件/目录本身直接拒绝(与 toybox 同策略,防 allow 区内放链指向外部)。
5. 单文件大小上限 5MB(超过不读,防内存)。

## 4. 注册与暴露

- `_TOOL_REGISTRY` 新 category `"fs"`,补 `examples`/`keywords`(硬性规则2);
  `trace_args: ["path"]`(路径在 allowlist 内,不敏感,落痕迹方便追问溯源)。
- **探针不覆盖 fs**(`get_probe_prompt` 只拼 info/desktop,不动):只经 loop 暴露,
  角色卡 `tool_categories` 加 `"fs"` 才可见。理由同 MCP——多步浏览本来就是 loop 行为,
  小模型探针单发读文件意义有限,真有需求下个 brief 把 fs_read 放进探针。
- 全局 `tool_loop.exclude_tools` 对 fs 同样生效。

## 6. 风险与不做什么

- **风险**:文件内容是不可信输入(与 web_search/MCP 结果同级),可能含提示注入文本;
  v1 接受现状,known-issues 登记。用户手填 allow_roots 是同意边界,不做二次确认。
- 不做:写入/删除/移动(永远不做进 fs 类);fs_search/grep(下个需求再说);分页读取;
  探针暴露;图片/PDF 解析(有 media_processor 的既有通道,不在此重复)。
