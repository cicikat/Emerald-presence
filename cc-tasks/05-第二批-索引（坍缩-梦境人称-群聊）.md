## B1. 语句坍缩：连续「现在，xxxxx」（生成端）

**现象**：聊天连着好几句以「现在，……」开头 = 语句坍缩 / mode collapse。茶茶设想加「随机种子噪音」扰乱多条语句的相似性。

**根因（已有诊断文档 `docs/hallucination-and-collapse-analysis.md` 第三节）**：
- 每轮是独立 `llm_client.chat()`，坍缩来自两点：① 模型在 `9_history` 层读到自己上一轮的近似回复 → 复读；② 连续两轮注入层几乎完全相同（activity / perception / 时间桶 / author note）+ 强角色约束 → 条件分布高度尖峰，`temperature=1` 也压不住。
- `frequency_penalty=0.3` 只在**单次生成内**生效，**跨轮无效**；跨轮没有任何「别和上一条同开头」的机制。

**能否用「随机种子噪音」实现**：纯文本扰乱有效但要对路——不是给 prompt 加乱码，而是：
1. **跨轮去同质**：把最近 2–3 条 assistant 回复的**开头词 / 句式**提取出来，作为一条软约束注入「避免再以『现在/此刻/……』开头，换一种起手」。这是直接对症的「文字层去相似」。
2. **注入层抖动**：每轮对 activity / author note / 时间桶等近乎不变的层做**小幅随机轮换或省略**，打破「连续两轮注入几乎相同」，降低条件分布尖峰。`core/author_note_rotator.py` 已有轮换雏形，可扩。
3. **采样参数**：`presence_penalty` 当前 reality 主调用是否真传到 API 要核（config 里有 0.4，但注意有「白名单过滤」注释，部分 preset 会被滤掉）；适度调 `presence_penalty`/降低重复，但这是辅助，**主力是 1 + 2**。

**关键文件**：`core/prompt_builder.py`、`core/pipeline.py`、`core/llm_client.py`、`core/author_note_rotator.py`、`config.yaml`。属生成端、较难，建议小步灰度（先上跨轮开头去同质，最低风险）。

## B2. 梦境旁白人称统一（渲染端转换）

**现象**：梦境旁白人称不统一。生成端以叶瑄视角陈述，叶瑄称用户「她」、自称「我」。茶茶设想：**渲染时**把旁白里的「她」→「你」、「我」→「他」。

**根因 / 现状（实读 `core/dream/dream_prompt.py`）**：
- 人称在生成端**全局锁死**（dream_prompt.py:19,259「叶瑄=他，用户=她」；:748「叶瑄用他/我，指称用户用她/你」）。输出是混合：对白带说话人标签 `她：…` / `叶瑄：…`，动作旁白在 `*…*` 内。
- 所以「她」既可能指用户（要转「你」），也可能出现在叶瑄的台词里指别的——**全局 replace 会误伤**。

**方案（别做朴素全局替换）**：
- **首选**：改生成契约，让旁白直接以「你」称用户（在 dream_prompt 的人称段把「向她」类改成「向你」），从源头统一，无需渲染转换。最干净、零误伤。
- **次选（茶茶原设想）**：渲染层**结构化**转换——只对「说话人标签」和 `*…*` 旁白跨度做 `她→你`，**跳过台词正文**；`我→他` 同理只在旁白跨度。需先定位梦境输出的渲染/分发点（`core/dream/dream_hud.py` / dream 回复 sink），按 span 切分后局部替换。比首选复杂且易漏，仅在不愿动生成契约时用。

**关键文件**：`core/dream/dream_prompt.py`、`core/dream/dream_hud.py`、dream 输出渲染处。

## B3. 群聊 prompt 注入：history 缺时间戳 + 无排序注释（低优先）

**现象**：群聊语义一片混乱、牛头不对马嘴。

**根因（实读 `core/memory/group_context.py` + `prompt_builder.py:633-651`）**：
- `append()` 存的 `timestamp` 只有 `"%H:%M"`（:55「只保留时分，省空间」）——**跨天即丢失日期**，无法判断先后。
- `get_recent()` 直接返回 `_load_raw()`，**不排序**。
- prompt_builder 注入（:638-645）按 `sender_name + content` 拼行，**没把时间戳带进注入文本**，也没有「按时间排序/这是历史回顾」的注释 → 模型读到一堆无时序、无身份锚的消息，自然错乱。

**方案**：
1. `append` 的 timestamp 存完整 `datetime`（或加日期），保留显示用短格式。
2. `get_recent` 按 timestamp 升序排序后返回。
3. prompt_builder 注入每行带 `[MM-DD HH:MM] {发言人}：{内容}`，并在层头加注释：「以下为群内最近消息，按时间先后排列，仅供参考语境，非对你的直接提问」。

**关键文件**：`core/memory/group_context.py`、`core/prompt_builder.py`（4_group_context 层）。低优先，独立小改。
