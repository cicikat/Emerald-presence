## 1. 现状盘点（已确认）

代码内（排除 tests、data/、__pycache__）：**`叶瑄` 86 处、`yexuan` 284 处**；`风谕`（用户名！）在 `core/dream/dream_prompt.py` 模板里多处出现。

**既有的两条去敏先例（本次统一到它们上，不新开路）：**

1. **ID 层**：`config.yaml → character.default` → `core/data_paths.py:19 _read_default_char_id()` → `_DEFAULT_CHAR_ID`，作为路径默认参数。启动兜底才是字面 `"yexuan"`。
2. **Prompt 层**：`char_name` 参数 + `.replace("叶瑄", char_name)` 补丁（如 `dream_prompt.py:298,392`），并已留 `TODO(方案B·多角色)` 标记。

问题：先例没有铺开——全仓还有大量 `char_id: str = "yexuan"` 默认参数（short_term/episodic/event_log/mood_state/mid_term/fixation/activity/dream 各模块）、模板里未被 replace 覆盖的字面量、以及协议层的 `yexuan_tension`、`yexuan_ai`。

## 2. 梦境流动（DREAM FLOW）后端接入

前端侧栏「梦境流动」当前读 `dream_state.flow_entries`，后端从未产出 → 永远显示前端三条固定 fallback 文案。拍板方案：**规则驱动、零额外 LLM 调用**。

### 2.1 数据

dream state（`core/dream/dream_state.py` 存储）新增：

```json
"flow_entries": [ {"ts": "<iso>", "kind": "scene_shift", "summary": "场景转入…"} ]
```

上限 10 条 FIFO；`/dream/enter` 时清空重开。

### 2.2 产出点（dream_pipeline 每轮收尾、已更新 scene/tension 之后）

规则生成，命中才追加（一轮最多 2 条，按下表优先级）：

| kind | 触发 | summary 模板 |
|---|---|---|
| `status_shift` | 状态迁移（进入梦境 / EXIT_REQUESTED / CLOSING / 挽留成功） | 「梦境正在成形」「醒来的边缘在靠近」「梦在慢慢消散」「他把你留了下来」 |
| `scene_shift` | `scene_state` 文本与上轮不同 | 「场景转入：{scene_state 前 20 字}」 |
| `tension_up` / `tension_down` | 张力较上轮 Δ≥0.15 | 「他的情绪张力在上升/回落」 |
| `anchor_new` | `symbolic_anchors` 新增 | 「新的象征浮现：{anchor}」 |

模板不写角色名（用「他」，与前端现有文案一致），天然免硬编码。

### 2.3 暴露

`GET /dream/state`（`admin/routers/dream.py:286`）返回体加 `flow_entries`（仅当前梦的条目；非梦境期返回空数组）。测试：状态迁移/场景变化/张力跳变各一条断言 + 上限裁剪断言。

## 3. 硬编码去除计划（分四阶段）

### 阶段 P0 —— Prompt 可见字面量（最高优先级，用户明确要求）

原则：**进入 LLM prompt 或展示给用户的文本，一律经 `char_name` / `user_name` 插值**。

1. `core/dream/dream_prompt.py`：`_D1_LUCID_AWARENESS`、`_D1_NON_LUCID_AWARENESS`、逃生协议、人称契约等模板中所有 `叶瑄` 改为 `{char_name}` 占位符，随处 `.format(char_name=...)`；删掉事后 `.replace("叶瑄", char_name)` 补丁（replace 只救了部分路径，D0 注释、D8 等仍有漏网）。清掉对应 `TODO(方案B·多角色)`。
2. **`风谕` 全部移除**：新增 `config.yaml → user.display_name`（`config.example.yaml` 同步，默认空），`core/config_loader.py` 提供 `get_user_display_name()`（空则回退「你」的无名称写法）。dream_prompt 模板改 `{user_name}` 插值。
3. 其余 prompt/文案产出文件逐个过（grep `叶瑄` 定位）：`core/activity/{reading,chess,gomoku}_companion.py`、`core/garden/manager.py`、`core/dream/{body_projection,distill_impression,dream_settings,body_tracker,impression_loader}.py`、`core/media_processor.py`、`admin/routers/{character,dream,settings_prompt_assets}.py`。现实侧已有 `character.name`（角色卡）可传；梦境侧沿用 dream_pipeline 已有的 char_name 来源。
4. 纯注释/docstring 里的 `叶瑄` 改为「角色」。

### 阶段 P1 —— `char_id` 默认参数统一

- `core/data_paths.py` 把 `_DEFAULT_CHAR_ID` 公开导出为 `DEFAULT_CHAR_ID`（语义不变：import 时冻结自 `character.default`）。
- 全仓 `char_id: str = "yexuan"`（约百处，short_term.py 一类）批量改为 `char_id: str = DEFAULT_CHAR_ID`（模块顶部 `from core.data_paths import DEFAULT_CHAR_ID`）。行为对默认部署零变化；多角色部署改 config 即全局生效。
- `"yexuan_inner"` 字面量仅允许存在于 `data_paths.py` 的 legacy 分支（白名单注释标明）；其他文件出现的一律改走 data_paths accessor。

### 阶段 P2 —— 协议字段/枚举更名（与前端联动）

| 项 | 现值 | 新值 | 兼容策略 |
|---|---|---|---|
| dream state / `GET /dream/state` | `yexuan_tension` | `char_tension` | 后端**双发**两字段 ≥1 个版本；内部读取处改新名；前端读新回退旧（client Brief 15 §G） |
| 五子棋对手枚举 | `yexuan_ai` | `character_ai` | 入参两值都收、归一化存新值；旧存档 load 时归一化；响应发新值 |

### 阶段 P3 —— 守门（防回流，"统一不要新开路"的执行面）

新增 `tests/test_no_hardcoded_character.py`：扫描 `core/ admin/ channels/ main.py` 源码，断言不出现 `叶瑄|风谕`；`yexuan` 仅允许出现在白名单位置（`data_paths.py` 的 `_read_default_char_id` 兜底与 legacy 分支、兼容双发/归一化代码，白名单写在测试里并注明到期条件）。跑进常规 pytest。

### 明确不做（记录在案）

- `data/` 运行时数据、`characters/` 角色卡与授权内容——是数据不是代码。
- 仓库名 `yexuan_memery`、移动端包名 `com.example.yexuan_memery`——改动影响安装与 git 历史，单独由用户拍板。
- tests 里作为 fixture 的 `yexuan`（跟随各阶段顺带改，不单列）。
