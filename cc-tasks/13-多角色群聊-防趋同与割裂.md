> 给 Claude Code 执行（仓库 `D:\ai\Emerald-presence`）。开工前读 `docs/stage.md`（+ `docs/specs/spec-10-group-chat.md` 如在）。
> **澄清**：这是「**前端多角色卡 × 用户**的群聊」（Stage 系统），**不是** QQ 群聊。
> cc-task 08/12 改的 `group_context.py` 是 QQ 群路径，与本单不同子系统——那两单仍有效（QQ 群也在用），算预防，但茶茶反馈的「多角色互相复读趋同/对话割裂」真正落点在这里。
> 改动集中在 **1 个文件**：`core/stage/context.py`（presence/transcript 渲染），低风险、自包含。

## 背景 / 根因（2026-06-29 实读）

茶茶反馈：多角色群聊**对话割裂**，且**角色经常互相读群聊上下文 → 复读、回答趋同**。

实读 Stage 链路定位到两条根因：

### 根因 1：同一轮里后发言的角色会读到先发言角色的回复 → 跨角色坍缩

`core/stage/runner.py`：
- Phase A（:93-117）每个候选角色依次 `_generate_and_append`，**每条回复立即 append 进共享 transcript**（:55），下一个候选 `generate_reply(stage, speaker_id, list(transcript), ...)` 拿到的是**已含前面角色本轮回复**的 transcript（:41）。
- Phase B（:120+）自主续聊，每条后重算，看到的 transcript 还在增长。

→ 后说话的角色**看到别人刚说的话**，加上各自 prompt 脚手架高度相似 → 倾向**附和/复述** → 回答趋同。这是「跨角色」版的语句坍缩（对应单角色的 B1-1/B1-2）。

### 根因 2：transcript/presence 注入层**零防趋同、零互动引导**

`core/stage/context.py`（仅 42 行）：
- `render_presence`（:8）只说「在场还有谁 + 你说的话别人看得到」，**没有**「保持你自己的性格/立场、别附和、要有区分度」。
- `render_transcript`（:20）把共享记录渲染成 `speaker：content`（自己=「你」、他人=名字、owner=owner），**没有**任何「别复述上面别人刚说的话」「从你的角度回应」的提示，也**没标**哪几条是本轮刚冒出来的。

→ 角色既无「保持差异」的约束，又无「真实互动（回应/反驳/补充别人）」的引导 → 要么趋同复读，要么各说各话（**割裂**：像并排独白，不像一场多人对话）。

这两层正是 `2.2_stage_presence` / `4.2_stage_transcript`（`core/prompt_builder.py:431-436`、`:309-310`，由 `context.py` 渲染喂入），改这里**只影响群聊、不碰单聊**。

## 实现（全在 `core/stage/context.py`）

### 1. `render_presence` 强化「保持自我 + 真实互动」

```python
def render_presence(stage: Stage, *, viewer_id: str) -> str:
    me = get_char_name(viewer_id)
    others = [get_char_name(c) for c in stage.roster if c != viewer_id]
    joined = "、".join(others) if others else "没有其他角色"
    return (
        "【群聊在场感】\n"
        f"现在你进入了群聊，你是「{me}」，在场的还有：{joined}。"
        "你说的话其他在场角色也看得到。\n"
        f"保持「{me}」自己的性格、说话方式和立场——"
        "看到别人怎么说，不要趋同、不要附和复述别人刚说过的话；"
        "有不同就表达不同，有自己的角度就说出来。"
        "你可以直接回应、反驳、补充或调侃其他角色（点名也行），"
        "像真实的多人对话那样彼此接话，而不是各说各的。"
    )
```

### 2. `render_transcript` 标记「本轮刚说的话」+ 防复读提示

让函数知道本轮 turn_id（runner 一轮共用一个 `_turn_id`），把本轮新发言标出来，并在末尾追加针对「上一条他人发言」的防附和提示：

```python
def render_transcript(
    stage, transcript, *, viewer_id, limit: int = 40,
    current_turn_id: str | None = None,     # 新增：本轮 turn_id
) -> str:
    lines, last_other = [], None
    for entry in transcript[-limit:]:
        if entry.speaker_id == "owner":
            speaker = "owner"
        elif entry.speaker_id == viewer_id:
            speaker = "你"
        else:
            speaker = get_char_name(entry.speaker_id)
            last_other = (speaker, entry.content)
        fresh = (current_turn_id is not None
                 and getattr(entry, "_turn_id", None) == current_turn_id
                 and entry.speaker_id not in ("owner", viewer_id))
        prefix = "（刚说）" if fresh else ""
        lines.append(f"{prefix}{speaker}：{entry.content}")
    text = "\n".join(lines)
    if last_other is not None:
        name, content = last_other
        snippet = content[:30]
        text += (
            f"\n\n（注意：{name} 刚说了「{snippet}…」。"
            "别复述或简单附和这句，从你自己的立场出发回应或另起话头。）"
        )
    return text
```

### 3. 把 `current_turn_id` 透传进来

`render_transcript` 的调用方（Stage runtime 组装 `stage_transcript` 文本处，搜 `render_transcript(` 的调用点，约在 `core/stage/runtime.py` / `views.py`）把本轮 `turn_id` 传进去。拿不到就传 `None`，函数自动退化为「不标记 fresh、但仍带末尾防附和提示」，**fail-safe**。

> 可选增强：若想更强，把「本轮其他角色已说观点的 gist 列表」单独拼一行「避免重复这些观点/措辞：…」——但上面的「针对最后一条他人发言」已能覆盖大多数复读场景，**建议先做 1+2+3，观察够不够**。

## 验收标准

1. 多角色同一轮：后说话的角色**不再附和/复述**前一个角色刚说的话，输出有各自性格区分度。
2. 对话不再「各说各话」：角色之间出现真实接话（回应/反驳/点名），群聊连贯感变好。
3. transcript 注入里本轮新发言带「（刚说）」标记、末尾带防附和提示；`current_turn_id` 缺失时不报错、退化正常。
4. 单角色私聊**完全不受影响**（这两个渲染函数只在 Stage 路径用）。
5. 既有 Stage 测试通过；如有 transcript 渲染相关断言，按新格式更新。
