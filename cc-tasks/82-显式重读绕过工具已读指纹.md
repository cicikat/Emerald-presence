# Brief 82 · 显式「再读一遍」绕过工具已读指纹（P2-1 落地）

> 裁决：DESIGN.md §十一 决策 7——用户显式意图优先于去重优化。known-issues P2-1 已关闭待此单。

## 现状

`core/memory/tool_read_log.py` 为 `persist=True` 工具（read_diary / read_watch /
read_toy_file / search_diary）记录已读指纹，重复读取被拦。用户显式说「再读一遍」时也被拦，
违背显式意图。

## 工单项（🟢 单项）

- 探针/意图侧识别显式重读短语（受控小集合：「再读一遍」「重新读」「再看一次」「重新看看」，
  常量表维护，不上 LLM 判断），命中时给本轮 `execute()` 的工具调用挂 `bypass_read_log=True`。
- `tool_read_log` 查询接口加 `bypass` 参数：bypass 时不拦，但**指纹照常记录/刷新**（只是不拦，
  不是不记）。
- 只对 `persist=True` 工具生效；短语必须与工具目标同轮出现（防「再读一遍」误触发无关工具）。

## 验收

- 同一文件第二次普通读取被拦（回归）；带显式短语 → 放行且指纹更新。
- 短语出现但本轮未触发任何 persist 工具 → 无副作用。
- `pytest -n auto`；文档：`docs/tools.md` 已读指纹一节补 bypass 说明，known-issues P2-1 移入已关闭表。
