# Brief 96 · 梦境创作补全:世界(文件夹)管理 + 剧本模式专用创作页

> 背景:真机复测(20260718)。现有「梦境设定」页只覆盖 sandbox 模式:世界书
> 条目和预设文本能编,但**世界本身(`characters/dream_worlds/{world}/` 文件夹)
> 无法新建/重命名/删除**;scenario(剧本)模式的剧本文件
> (`data/dream/scenarios/*.yaml`)**后端连 CRUD 端点都没有**,面板自然没有
> 创作入口。与 Brief 95 **可并行**。施工前先读 `docs/dream.md` 厘清三模式
> (sandbox / scenario / mirror)与剧本 YAML 的字段结构(参考
> `data/dream/scenarios/prison_demo.yaml`)。

## 1. 🟡 世界文件夹管理端点 + UI

- 后端 `admin/routers/dream.py` 补三个端点(admin scope,路径全部走
  `core/sandbox.get_paths()` 相关 accessor,不许硬拼):
  - `POST /dream/worlds`:新建世界(建文件夹 + 最小骨架文件,骨架内容参照
    `_default` 世界的文件构成);世界名校验(合法文件名字符集、与现有世界
    不冲突、保留名 `_default`/`reality_derived` 拒绝)。
  - `PUT /dream/worlds/{world}/rename`:重命名(文件夹改名;先核实
    `anchor_weights.json` 或其他索引/引用是否存世界名——有则同步更新,
    正在被进行中的梦引用时拒绝并提示)。
  - `DELETE /dream/worlds/{world}`:删除(二次确认由前端做;`_default` 与
    `reality_derived` 拒删;删除前若为当前梦境默认世界,重置为 `_default`)。
- 面板梦境设定页世界下拉旁加「新建 / 重命名 / 删除」按钮,接上述端点。
- 注意 `characters/dream_worlds/` 在 .gitignore 内(用户数据),行为正确,
  但**新建骨架的模板文件必须来自 tracked 的模板源**(如 `_default` 随仓
  发布——若 `_default` 实际不在版本库,先把一份中性模板放进 tracked 位置,
  fresh 安装才有东西可复制,这一点必须实测 release 包环境)。

## 2. 🟡 scenario 剧本 CRUD 端点

- `admin/routers/dream.py`(或独立 router)补:
  `GET /dream/scenarios`(列表)、`GET /dream/scenarios/{id}`(读)、
  `POST /dream/scenarios`(新建)、`PUT /dream/scenarios/{id}`(改)、
  `DELETE /dream/scenarios/{id}`(删)。
- 存储沿用 `data/dream/scenarios/{id}.yaml`,路径走 sandbox accessor;
  YAML 结构校验按剧本加载器的实际 schema(以 `prison_demo.yaml` +
  加载代码为准),校验失败返回具体字段错误而不是 500。
- 正在进行中的梦引用的剧本拒改拒删(提示「梦醒后再编辑」)。

## 3. 🟡 梦境设定页重构:第一行选模式 → 调对应创作栏

- 页面第一行加模式选择(sandbox / scenario / mirror 三选一,默认 sandbox):
  - **sandbox**:显示现有创作栏(世界选择+§1 管理按钮、世界书、预设文本),
    行为零回归。
  - **scenario**:显示剧本创作栏——剧本列表 + 新建/编辑/删除(接 §2);
    编辑器 v0.1 用「表单化关键字段 + 原始 YAML 文本区」双栏即可,不做
    完整可视化编排。
  - **mirror**:无创作物(倾向材料由 hidden_state 自动生成),显示只读说明
    (一段话讲清 mirror 是什么、为何不可编辑),避免用户以为功能缺失。
- 模式选择只影响本页显示哪套创作栏,**不改变入梦时的 dream_mode 选择逻辑**
  (入梦模式在哪选就还在哪选,两处如有联动按现状保持)。

## 4. 🟢 文档同步

- `docs/dream.md` 补:世界管理端点、scenario CRUD 契约、设定页三模式结构;
  README 功能列表如提及梦境创作,措辞对齐「三模式」。

## 验收

- 面板全流程不碰文件系统:新建世界→写世界书→重命名→删除,均正常且保留名
  受保护;release 包 fresh 安装环境新建世界骨架可用。
- 新建剧本→入梦选 scenario 跑通一场→改剧本被拒(梦中)→梦醒后可改可删。
- sandbox 现有创作功能零回归;mirror 页说明可见;`pytest -n auto` smoke 通过。
