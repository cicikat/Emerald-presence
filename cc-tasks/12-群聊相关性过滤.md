## 背景（现状已确认）

一期把群消息按时间排好、加了「这是语境非提问」的注释，缓解了「牛头不对马嘴」。但**注入的是最近 N 条全量**——群里灌水、跑题、与当前对话无关的消息照样塞进 prompt，仍会稀释语义、诱导模型去接无关的话。茶茶要的「按相关性过滤」就是这一层。

实读：

- `core/prompt_builder.py` 4_group_context（约 :636-663）遍历这些消息拼 `[时间] 发言人：内容`，无任何相关性筛选。
- `build_prompt` 手上有**当前用户消息**与**话题 tags**（`tags` 参数）——这是打相关性分的现成依据。

## 实现

### 1. 相关性打分 helper（放 `core/memory/group_context.py` 或 prompt_builder 内）

对每条群消息算一个分，综合三项：
```python
def _relevance_score(msg: dict, *, query_text: str, tags: set[str], char_name: str) -> float:
    text = str(msg.get("content") or "")
    score = 0.0
    # (a) 与当前用户消息的关键词重叠（简单 char-ngram / 分词交集即可，别引重依赖）
    score += _keyword_overlap(text, query_text) * 1.0
    # (b) 命中当前话题 tag 关键字
    if any(t.split(".")[-1] in text for t in tags):
        score += 0.5
    # (c) @ 或点名角色（被叫到名字，强相关）
    if char_name and char_name in text:
        score += 0.8
    return score
```
`_keyword_overlap` 用最轻的实现（中文按 2-gram 交集 / 去标点后子串命中计数）；**不要**引入分词大库。fail-safe：算分异常该条记 0。

### 2. 过滤策略（在 4_group_context 注入前）

既要相关、又不能丢掉「最新的连续语境」：
```python
recent = group_context                      # 已按 ts 升序
keep_latest = recent[-_GC_KEEP_LATEST:]     # 末尾 N 条永远保留（保证连贯，默认 3）
scored = sorted(
    recent[:-_GC_KEEP_LATEST],
    key=lambda m: _relevance_score(m, query_text=user_text, tags=tags, char_name=character.name),
    reverse=True,
)
relevant = [m for m in scored if _relevance_score(...) > _GC_MIN_SCORE][:_GC_TOP_K]
# 合并后按 ts 复排，保持时间顺序注入
chosen = sorted({id(m): m for m in (relevant + keep_latest)}.values(),
                key=lambda m: float(m.get("ts") or 0))
```
- 参数走 config（`memory.group_context_*`），给默认：`_GC_KEEP_LATEST=3`、`_GC_TOP_K=5`、`_GC_MIN_SCORE=0.3`。
- **私聊不受影响**（group_context 为空时整段跳过，一期已是）。
- 用 `chosen` 替换原来直接遍历 `group_context` 的那段，其余注入格式（`[时间] 发言人：` + 注释）不变。

### 3. 注释微调

层头注释补一句，告诉模型这是**筛过的相关片段**：
> 「以下是群里与当前对话较相关的最近消息（已按相关性挑选、按时间排列），仅供理解语境……」

## 验收标准

1. 群里混入明显无关的灌水消息时，注入的 4_group_context **不再包含**低相关项（用 admin 的 prompt 层检视页确认）；与当前话题/被点名/最新连续语境相关的仍在。
2. 末尾最近 N 条始终保留（不会因为「不相关」把刚刚的对话上下文也滤掉，避免断裂）。
3. 注入仍按时间升序、格式不变；私聊不注入该层。
4. 打分异常/空输入不崩（fail-safe 记 0）。
5. 参数可经 config 调（top_k / keep_latest / min_score）。

---

## 备注

- 打分是**轻量启发式**，不引向量/分词重依赖；够用即可。若以后想更准，可复用语义向量库给群消息做语义相似度（届时另议）。
- 与一期（时间戳+排序）叠加：先排序、再筛相关、再复排时间——顺序别搞反，否则注入乱序。
