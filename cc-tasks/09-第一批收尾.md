## 3. 补 `global_proactive_min_gap_seconds` 配置项

触发器节律 Part B 已实现，但 `config.yaml` 的 `scheduler:` 块**没加这个 key** → `_global_proactive_gap_ready()` 走默认 45 分钟、茶茶调不了。

在 `config.yaml` 的 `scheduler:`（约 :274）补一行：
```yaml
scheduler:
  # ... 现有项 ...
  global_proactive_min_gap_seconds: 2700   # 主动消息全局最小间隔(秒)；45分钟。想更克制就调大
```
**验证**：改成一个很小的值（如 60）跑一轮，确认主动消息间隔随之变化；改回 2700。

## 4. `reminders.py:77` 人称残点（可选，低优先）

第一批人称统一漏了一处（不在原清单内）：
