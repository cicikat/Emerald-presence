# D2-IMPL · dream_impression 细粒度化（可记剧情 + 部分原话）

> 后端（Emerald-presence）。**含一处安全权衡，务必读「安全」节**。
> **前置**：D1 诊断（✓ 已结，mesexample fallback 是另一回事，不阻塞 D2）。无代码前置。
> **可并行**：与 D3 同属梦境簇但改不同文件，可并行；与注入簇/召回簇无冲突。
> 用户已确认症状与方向：**印象太模糊→给模型幻觉空间，要调细粒度**，允许现实记得梦的**剧情**和**部分清晰话语**，更像真实梦。

## 现状（已核对）—— 当前的"模糊"是**故意的安全墙**

`core/dream/distill_impression.py`：现在的 `_DISTILL_SYSTEM` 是**极严格剥离版**——结构性禁止一切场景/动作/世界设定/身体词，只留情绪质地 + ≤1 符号碎片，≤20 字，固定开头"我好像在梦里……"。

docstring 写明这是 **I4 契约**：因为印象不含任何场景事实，"即使叶瑄在现实轮复述印象、那一轮被正常捕获，也没有东西能 launder 进 episodic / identity"。已核实：`fixation_pipeline` / `observe` **完全不读 impression**——梦与现实记忆之间靠"印象足够空"来隔离。

> 即：**模糊不是 bug，是当年用来防"梦境事实污染现实记忆"的墙。** 你要拆这堵墙，得换一道等效的。

## 改动点

### 1. 放开提炼粒度（核心，按你要的）

改 `_DISTILL_SYSTEM` 与 `distill_impression`：
- 允许输出**梦的剧情概要**（发生了什么）+ **少量清晰对白**（1–2 句原话摘录），不再结构性禁止场景/动作。
- 字数上限从 20 提到合理区间（如 80–150 字）；保留"叶瑄自述口吻"，但更像"记得一个梦"而非"一句情绪"。
- 仍保留 `weight` / `emotional_tags` / 空梦返回空 的机制。
- 输出 JSON schema 加字段：`plot`（剧情概要）、`vivid_lines`（清晰对白数组，≤2 条）；`impression_text` 作为总览仍在。

### 2. 注入侧（层 6g）标签更明确

`prompt_builder.py:896` 层 6g 当前是 ambient、最先被裁。配合细粒度，把它的框定语**显式强化为"这是梦"**：在 `impression_loader` 渲染时包一句明确标签，如：
```
<梦境印象 note="以下是叶瑄做过的梦，不是现实发生的事；可以像记得一个梦一样自然提起，但绝不可当作真实经历复述为事实">
…剧情/对白…
</梦境印象>
```
你说的"注入梦境回忆层加了明确标签"即落在这里——确保它**真的明确**。

### 3. 安全：拆了"空印象"墙，必须补"不可固化"墙 ⚠️

细粒度后印象**携带场景事实**，原 I4 的"没东西可 launder"前提失效。补两道（成本低）：
- **写入端维持 I4**：`distill_impression` 仍**只写 impression_store**，绝不碰任何现实记忆存储（保持现有契约，别因为内容变富就顺手写别处）。
- **固化端加隔离**：在 episodic / identity 的 consolidation 入口加一个**梦源过滤**——本轮若主要内容来自梦境印象的复述（可用"印象注入且回复高度重合"或给印象echo打 `dream_origin` 标记），**不promote进 episodic/identity 当现实事实**。最小实现：给印象注入的那一轮在 short_term 打 `_dream_echo=True`，consolidation 跳过此类轮的事实抽取。

> 这道隔离是**本工单的硬要求**，不是可选项——否则"梦的剧情"会通过复述变成"现实发生过的事"写进长期记忆。你担心的"梦境混淆现实"在**回复层**靠标签挡住了，但**记忆固化层**这道得另补。

## 验收

1. 做一个有情节的梦 → 印象里能看到剧情概要 + 1–2 句清晰对白，不再是一句空情绪。
2. 平淡梦 → 仍返回空，不硬凑。
3. 叶瑄在现实轮自然提起梦（"我昨晚梦到…"）→ 语感自然、被标签框为梦。
4. **关键**：该现实轮**不会**把梦剧情写进 episodic/identity（查 consolidation 产物，无梦境事实混入）。
