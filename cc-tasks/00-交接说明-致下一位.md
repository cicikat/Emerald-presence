## 二、我拍板时守的几条惯性(建议延续)

1. **一切新能力挂开关、默认关、崩坏可一键回退。** 这个系统的核心资产是角色的"人味"
   和五层记忆的干净,任何 agent 能力都是外挂,外挂永远要能摘。
2. **危险面收窄成白名单,fail-closed;体验面 fail-open。** 工具 origin、fs 守卫、
   deny 底线集是前者;独白失败跳过、痕迹记录失败不影响主流程是后者。
3. **记忆核心不外包。** MCP/外部工具只进"感知和行动",不碰记忆五层。外接记忆库
   会绕过固化链裂成两套真相,29 号文档里写了理由,别被"接个记忆 MCP 很方便"诱惑。
4. **loop 中间态一次性丢弃,跨轮记忆只走 action_trace。** 工具调用消息进 history 会
   触发风格反馈坍缩(repo 的老伤,见 `_sanitize_assistant_message` 和 F 系列 issue)。
5. **provider 方言不做逐家适配,统一 OpenAI-compat 网关 + preset 声明 + extra_body 逃生舱。**
   代码里出现"if provider == claude"就是走错路的信号。
6. 每份文档先做**现状盘点**且标注 file:line——cc 拿到的必须是核实过的事实,
   不是我的想象。你接手写文档时建议保持,这个 repo 的 AGENTS.md 任务路由表很好用,先读它。
