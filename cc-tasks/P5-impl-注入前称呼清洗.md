# P5-IMPL · 注入前称呼/口吻清洗（填 P1 的 normalize 钩子）

> 后端（Emerald-presence）。**小修**。
> **前置**：P1 骨架（✓ 已落地，`_normalize_injection` 空壳 seam 在 `prompt_builder.py:197`，已对所有 system 层接线于 :1126-1128）。
> **可并行**：与 P2 / P3 / P4 / D 系列互不冲突。**只需往钩子里填规则**，不碰结构。

## 现状（已核对）

注入文本里散落第三人称系统口吻「用户 / user」，会把角色语感带偏。出现处（举证，非穷举）：
- 层 3 关系：`该用户是你的{role}`、`<与用户关系>【与该用户的关系】`（`prompt_builder.py:473/487`）
- 层 5 画像：`<用户概况>【关于这个用户】`（:674）
- 层 5.1：`<用户客观信息（跨角色通用…）>`（:688）
- 层 6a：`关于用户的长期观察…`（:729）
- mid_term / user_identity 渲染文本里的「用户」。

`_normalize_injection(text, char_name)` 现在是恒等返回，P5 在此填规则。

## 目标

注入给模型的 **system 层正文**里，把指代「那个人」的第三人称「用户/user」统一替换为「**她**」，集中在一个钩子，不再逐处手改；**不碰真实对话**（history 是 user/assistant role，seam 已天然排除）。

## 改动点（全部在 `_normalize_injection` 内）

### 1. 称呼替换规则

- 正文里的指代性「用户」→「她」：如「该用户是你的恋人」→「她是你的恋人」、「关于这个用户」→「关于她」、「关于用户的长期观察」→「关于她的长期观察」。
- 英文 `user`（独立词，大小写）同理 → 「她」。

### 2. 保护项（别误伤）

- **XML 定界标签名不动**：`<与用户关系>`/`<用户概况>`/`<用户客观信息>` 这类标签是给系统配平用的，替换其内部标签名会破坏 :1113 的完整性检查。**规则只作用于标签之间的正文，不改 `<...>` 标签本身**——用「替换时跳过尖括号内 token」或「只替换中文方括号【…】与正文中的『用户』」实现。
- 「用户客观信息（跨角色通用）」这种**有意区分"非角色记忆"语义**的标题——保留语义，可改成「她的客观信息（跨角色通用，非角色记忆）」，别丢掉"非记忆"提示。
- 工具/内部字段名、`user_id` 等标识符不动（只作用于面向模型的自然语言正文）。

### 3. mid_term / identity 源头可选顺带

mid_term `format_for_prompt` 与 user_identity 渲染若在文本里硬写「用户」，可在源头改为「她」或留给 seam 统一兜——**优先让 seam 兜**（单点维护）。

## 验收

1. 构造含层 3/5/5.1/6a 的 prompt，断言注入正文里无裸「用户/user」指代，均为「她」。
2. 定界标签 `<...>` 完整、:1113 完整性检查无新 WARNING。
3. history（真实对话）原文不变。
4. 「非角色记忆」类语义未丢失。
