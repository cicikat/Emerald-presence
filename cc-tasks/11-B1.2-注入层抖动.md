# CC 任务：B1-2 注入层抖动（坍缩二期）

> 给 Claude Code 执行（仓库 `D:\ai\Emerald-presence`）。开工前读 `docs/hallucination-and-collapse-analysis.md`。
> 这是「语句坍缩」的二期。一期 B1-1（`9_anti_repeat` 跨轮去同质）已上；本单从**注入层**侧再降一档坍缩概率。**小步、低风险。**

---

## 背景（现状已确认，2026-06-29 实读）

坍缩第三因（诊断文档第三节）：**连续两轮的注入层几乎完全相同** → 条件分布尖峰 → 即便 `temperature=1` 也复读。一期治了「读到自己上一轮回复」，本单治「注入层逐轮雷同」。

实读现状：
- `core/author_note_rotator.py`：author note（注入层 11）**每 30 分钟**或**同一条连用超 `_SWITCH_INTERVAL_ROUNDS = 6` 回合**才切换（:15）。也就是说**密集对话里同一条 author note 会连用 6 回合**——这 6 轮里第 11 层逐字相同，是坍缩的稳定贡献源。
- `2.6_activity`（`core/prompt_builder.py:489`）只在**对话开头**注入（`should_inject = not bool(history)`），进行中不重复 → **不是**逐轮雷同源，**本单不动它**。

> 所以 B1-2 的有效落点是 **author note 的切换节奏**：让第 11 层在连续轮里更早产生变化。

---

## 实现

### 1. 缩短连用回合阈值 + 每轮小概率提前切换（`core/author_note_rotator.py`）

`_should_switch(state)`（:48）当前条件：30 分钟到点 **或** 连用 > 6 回合。改为：
- 把 `_SWITCH_INTERVAL_ROUNDS` 从 `6` 降到 `2`（同一条最多连用 2 轮）。
- 再加一个**每轮抖动**：未到硬阈值时，也以小概率（如 `0.35`）提前切换，使逐轮不可预测、不再死板 6 轮一换。
```python
import random
_SWITCH_INTERVAL_ROUNDS = 2
_JITTER_SWITCH_PROB = 0.35   # 未到阈值时每轮提前切换概率

def _should_switch(state: dict) -> bool:
    # ... 原有的 30 分钟到点判断保留 ...
    rounds = state.get("rounds_on_current", 0)
    if rounds >= _SWITCH_INTERVAL_ROUNDS:
        return True
    if rounds >= 1 and random.random() < _JITTER_SWITCH_PROB:
        return True
    return False
```
- `_pick_note`（:73）已有「`days_since <= 1` 的近用条目 forbidden」+ 加权随机，**天然保证换的是不同风格**，不会换到刚用过的。切换更频繁后这条更要生效——确认 forbidden 逻辑在小池子下不会把所有候选都禁掉（池子太小时要有回退：forbidden 后无候选则放宽）。

### 2.（可选）近静态层措辞/顺序轻抖

对其余逐轮几乎不变、且会进生成的软层（如情绪软提示 `core/mood_text.py` 的固定措辞、时段桶），可加**同义措辞轮换**或**层内顺序轻微打乱**，进一步去尖峰。**收益边际、风险略高**，建议**先只做第 1 步**，观察坍缩是否缓解，再决定要不要做这步。不做则本单仅含第 1 步。

## 验收标准

1. 密集多轮对话中，author note（层 11）**不再连续 6 轮逐字相同**；2 轮内必变、且变化点带随机不固定。
2. 切换后选到的是**不同**条目（不回退到刚用过的；小池子下不报错、有回退）。
3. 主观：配合一期 `9_anti_repeat`，连续多轮的开头/句式复读明显减少。
4. author note 池为空/缺失时回落默认池逻辑不回归（`_load_pool` 的 fallback）。
5. 不破坏既有测试；若有 author_note 相关测试，更新其对 `_SWITCH_INTERVAL_ROUNDS` 的断言。

## 备注

- 这是**降低坍缩概率**的叠加项，不是开关式根治。一期（跨轮去同质）+ 本期（注入去雷同）两条腿一起，体感才稳。
- 调参可后续观察：`_SWITCH_INTERVAL_ROUNDS` 与 `_JITTER_SWITCH_PROB` 都可按体感微调，换太勤会让人设提醒显得跳。
