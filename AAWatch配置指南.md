# Watch 配置指南

通过 iPhone 捷径（Shortcuts）将 Apple Watch 的心率和睡眠数据实时推送给机器人，触发心率关心、睡眠感知等主动消息。

---

## 前置条件

- iPhone + Apple Watch（watchOS 7+）
- 机器人运行在你的电脑上，且**电脑和手机在同一局域网**（或通过公网内网穿透可访问）
- 已启动机器人（`python main.py`），管理面板能正常打开

---

## 第一步：确认接口地址

机器人的 Watch 接收接口地址是：

```
http://你的电脑局域网IP:8080/watch/event
```

查看电脑局域网 IP：Windows 打开命令提示符，输入 `ipconfig`，找到 `IPv4 地址`，通常是 `192.168.x.x`。

> 如果手机不在家庭网络内（比如出门了），需要用 ngrok 等工具做内网穿透，把 8080 端口暴露到公网，将地址替换成穿透后的 URL。

---

## 第二步：配置 watch_secret（可选但推荐）

在 `config.yaml` 的 `scheduler` 节添加：

```yaml
scheduler:
  watch_secret: "你自己设的一串字符"   # 例如 WatchKey2024
```

设置后，捷径请求 URL 需要带上 `?secret=你自己设的一串字符`，防止局域网内其他设备误触发。不设置则不校验。

重启机器人后生效。

---

## 第三步：创建心率捷径

在 iPhone 上打开**捷径（Shortcuts）**，新建捷径，按以下顺序添加动作：

1. **获取我的健康样本**
   - 类型：心率
   - 来源：Apple Watch
   - 时间范围：最近 1 分钟
   - 限制：1 个结果（最新）

2. **获取"健康样本"的数值**（从上一步结果取心率数值）

3. **获取词典内容**（构建请求 body）
   - 添加两个键值对：
     - `type` → 文本 → `heart_rate`
     - `value` → 数字 → 上一步的数值

4. **获取 URL 的内容**（发送 POST 请求）
   - URL：`http://192.168.x.x:8080/watch/event?secret=你的secret`（没设 secret 就去掉 `?secret=...`）
   - 方法：POST
   - 请求体：JSON → 上一步的词典

完成后给捷径命名，例如"推送心率"。

**设置自动化触发：**

打开捷径 → 自动化 → 新建个人自动化：
- 触发条件：**心率高于 100**（或你想要的阈值）
- 动作：运行捷径"推送心率"
- 关闭"运行前询问"

> 也可以设置"心率高于 120"再建一个，对应不同的关心力度。

---

## 第四步：创建睡眠捷径

新建捷径，按以下顺序添加动作：

1. **获取我的健康样本**
   - 类型：睡眠分析
   - 来源：Apple Watch
   - 时间范围：最近 12 小时
   - 按开始时间排序，不限数量

2. **获取"健康样本"中的开始时间**（取所有样本的入睡时间列表）

3. **获取"健康样本"中的结束时间**（取所有样本的起床时间列表）

4. **将文本格式化为 HH:mm**（分别格式化入睡时间和起床时间列表，每行一个）

5. **获取词典内容**（构建请求 body）
   - `type` → 文本 → `sleep_end`
   - `sleep_start` → 文本 → 格式化后的入睡时间（多条用换行拼接即可）
   - `sleep_end` → 文本 → 格式化后的起床时间

6. **获取 URL 的内容**（发送 POST 请求）
   - URL：`http://192.168.x.x:8080/watch/event?secret=你的secret`
   - 方法：POST
   - 请求体：JSON → 上一步的词典

**设置自动化触发：**

- 触发条件：**睡眠阶段结束**（或"起床提醒"）
- 动作：运行捷径"推送睡眠"
- 关闭"运行前询问"

> 机器人收到睡眠数据后会等待 1 分钟合并阶段数据，再发送早安消息。如果你有午睡习惯，可以在捷径里加判断：只在上午 5～11 点运行。

---

## 验证推送是否成功

1. 手动运行捷径（直接点击运行按钮）
2. 打开管理面板 `http://127.0.0.1:8080` → Watch 状态
3. 如果能看到最近一次事件的时间和数值，说明推送成功

也可以在机器人日志里搜索 `[watch]` 关键字确认接收情况。

---

## 常见问题

**捷径运行成功但机器人没收到**
- 检查手机和电脑是否在同一 Wi-Fi
- 检查电脑防火墙是否放行了 8080 端口
- 用手机浏览器访问 `http://电脑IP:8080`，能打开管理面板说明网络通

**收到数据但没有发消息**
- 心率触发有阈值（>100 或 >120）和冷却时间（30分钟/1小时），正常心率不触发
- 睡眠触发在早上 12 点前才发早安，下午推送会被忽略
- 检查 `config.yaml` 的 `scheduler.enabled` 是否为 `true`

**出门时推送失败**
- 需要内网穿透，推荐 [ngrok](https://ngrok.com)：`ngrok http 8080`，把生成的 `https://xxx.ngrok.io` 替换捷径里的地址
