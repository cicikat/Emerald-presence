# 安全补丁验收 Checklist(通用模板)

> 适用:任何「收窄执行/写入权限、加 grounding、堵注入面」类补丁。
> 用法:起任务时让 CC 读此表;交活时逐条打勾。

## 阶段闸门
- [ ] RECON ONLY 阶段完成,禁止编辑,grep 关键词清单已回报
- [ ] 调用链是「实证 trace」而非「推断」(贴出真实链路)
- [ ] 范围裁决已拍板,不在范围内的冲突点已显式记录(非默默改)

## 不变量
- [ ] 核心不变量 ≤5 条,每条有对应代码落点
- [ ] 唯一触发/写入真值已定义(哪个变量是可信源)
- [ ] 不可信片段(media/tool result/memory/dream/authored)已标记隔离

## fail-closed(逐条验)
- [ ] 缺失 provenance/origin → 拒绝(且**无 permissive 默认**;漏传当场报错或拒绝+log)
- [ ] 可信源为空 → 跳过,不降级执行
- [ ] 非真实来源(scheduler/sensor/dream)→ 全跳过
- [ ] dangerous 操作无显式确认 → 拒绝
- [ ] 校验抛异常 → 视为「未通过」跳过,绝不「出错照常执行」

## 闸门实现
- [ ] 收口在单点(一个 execute/一个 guard),不散落多处
- [ ] 白名单是 allowlist 不是 denylist
- [ ] 幂等/去重 key 对无参情况稳定(确认不会每次生成新 key)
- [ ] 时间窗覆盖真实反射链路时延(别只看理论最小值)

## 测试
- [ ] 每个不变量有对应正向 + 反射/绕过的反向用例
- [ ] 金标准回归用例(真实观测到的 bug)单列且复现→修复
- [ ] fail-closed 路径有断言(拒绝/TypeError/零副作用)
- [ ] 在隔离数据(run_test.py)下跑,不污染生产 data/

## 收尾
- [ ] 提交粒度可单独回滚(闸门/守卫/测试分 commit)
- [ ] 文档同步(或明确「无需更新:理由」),不留给下轮 hook 翻账
- [ ] 不在范围内的东西确实没被顺手重构(对照禁改清单)
